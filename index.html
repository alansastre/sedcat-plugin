Contexto

SEDCAT son las siglas de Sistema Experto Difuso para la medición de CAlidad en el Testing.

Sedcat consiste en un plugin para la plataforma de gestión de calidad SonarQube cuya intención es la de aportar mayor información acerca de la calidad de las pruebas unitarias de un determinado software programado en lenguaje Java. SonarQube es una plataforma opensource pensada para trabajar en un entorno de integración continua en el que el código desarrollado se integra, prueba y analiza a diario con el fin de detectar fallos y estimar su calidad lo antes posible.

El contexto sobre el que este plugin trabaja está compuesto por dos ámbitos, siendo el primero de ellos el de las pruebas de software y el segundo el de medición y análisis.

Las pruebas del software consisten en otro software creado con la intención de probar el primero. Su objetivo es detectar fallos en el software que compone un determinado producto con el menor coste posible y verificar que el producto se está construyendo correctamente, es decir, que se comporta de la manera deseada. Sedcat trabaja en el ámbito de las pruebas unitarias para el alcance de la primera versión, postergando otros ámbitos de las pruebas (integración, aceptación, sistema) a futuras versiones.

La medición y análisis consiste en un área de proceso relacionada con la calidad dentro de las organizaciones de desarrollo de software. Trabaja principalmente a 3 niveles, siendo el más general el proceso o conjunto de actividades que se llevan a cabo en la organización, después está el proyecto donde la medición facilita las estimaciones y por último el producto. El producto software hace referencia a los artefactos o entregables generados durante el proceso de desarrollo y para los que existe gran variedad de métricas que aportan información útil. Sedcat trabaja en este último nivel leyendo métricas a nivel de producto para hacer sus cálculos y ofrecer nuevas métricas de salida que aporten mayor información.

El objetivo de la medición y el análisis es la búsqueda de calidad y se fundamenta en la cita escrita por el físico y matemático británico Lord Kelvin en 1885, que dice de manera resumida lo que no se puede medir no se puede mejorar.

Objetivos

Los objetivos de sedcat pasan por ofrecer una solución al problema que supone la escasez de métricas en el software de pruebas. Esto se debe a que si bien existe gran variedad de métricas del producto software en SonarQube pocas están relacionadas específicamente con el software de pruebas. Principalmente existen 2 cuyo uso está extendido y son ampliamente conocidas. Una de ellas es el éxito o el porcentaje de casos de test que se ejecutan correctamente de principio a fin sin fallar. La otra es el porcentaje de cobertura o porcentaje de código que las pruebas han alcanzado. Es importante matizar que el código alcanzado no quiere decir que haya sido probado, un test puede perfectamente cubrir gran parte de código pero no probar absolutamente nada.

El objetivo principal de sedcat es suplir dicha escasez proporcionando nuevas métricas relacionadas con la calidad de las pruebas unitarias. Esto implica también la consecución de los siguientes objetivos:

  • Facilitar la toma de decisiones en el proceso de desarrollo software al proporcionar mayor información sobre el estado de las pruebas. El usuario no tiene que revisar el código de las pruebas continuamente para ver qué debe mejorar, el propio plugin le aconseja.
  • Agilizar las etapas de verificación en el proceso de desarrollo de software. Al facilitar la toma de decisiones en las pruebas se acortan los tiempos en la etapa de verificación.
  • Ahorrar en deuda técnica (costes medidos en horas/persona) en las pruebas, al lograr pruebas de mayor calidad éstas son más fáciles de mantener.
